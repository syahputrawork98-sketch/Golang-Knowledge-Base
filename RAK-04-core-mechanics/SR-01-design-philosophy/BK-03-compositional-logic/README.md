# BK-03: Compositional Logic

Buku ini membahas logika komposisi yang menjadi ciri khas Go: kepuasan interface secara implisit dan pendekatan composition over inheritance.

## Struktur

### [CH-01_ImplicitSatisfaction](./CH-01_ImplicitSatisfaction/)
Cara interface dipenuhi tanpa deklarasi eksplisit dan dampaknya terhadap desain sistem.

### [CH-02_CompositionVsInheritance](./CH-02_CompositionVsInheritance/)
Kenapa Go lebih mendorong komposisi daripada pewarisan kelas tradisional.

## Boundary

- fokus pada logika desain yang membuat sistem Go lebih modular dan fleksibel;
- membantu pembaca melihat komposisi sebagai fondasi, bukan workaround;
- bukan tempat utama untuk pattern arsitektur tingkat aplikasi seperti middleware atau dependency inversion.
