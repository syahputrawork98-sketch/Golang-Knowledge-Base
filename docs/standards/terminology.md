# Panduan Terminologi (Go Edition)

Menangkap efisiensi dan ketangguhan system programming melalui bahasa yang tepat sesuai dengan **Unified Gold Standard**.

## 1. Aturan Penulisan Istilah
- **Efficiency Focus**: Gunakan kata kerja yang mencerminkan performa (e.g., *Allocated*, *Orchestrated*, *De-referenced*, *Multiplexed*).
- **Spec-Sync**: Seluruh istilah teknis merujuk pada standar resmi Go.

## 2. Senior vs Basic Terms
Gunakan terminologi profesional untuk akurasi teknis:
- **Channels** (bukan sekadar "pengirim pesan").
- **Goroutines** (bukan sekadar "thread").
- **Escape Analysis** (bukan "pindah memori").
- **Statically Typed** (bukan "tipe tetap").

## 3. Hierarchical Terminology (6-Level Standard)

| Term | Level | Analogi | Deskripsi |
| :--- | :--- | :--- | :--- |
| **Root** | 1 | Hub | Pusat navigasi utama repositori. |
| **RAK (Rack)**| 2 | Domain | Pengelompokan besar area ilmu (e.g., Golang). |
| **SR (Sub-Rack)**| 3 | Track | Jalur spesifik di dalam domain (e.g., Learning Go). |
| **BK (Book)** | 4 | Koleksi | Kumpulan bab yang membentuk satu modul utuh. |
| **CH (Chapter)**| 5 | Materi | Unit pengerjaan materi inti (Stage 1-4 PPM). |
| **SEC (Section)**| 6 | Detil | Granulitas terdalam untuk membedah sub-materi. |
